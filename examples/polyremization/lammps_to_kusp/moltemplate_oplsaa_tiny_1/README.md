# Moltemplate OPLS-AA Tiny Step 1

This folder is the first format-only checkpoint for importing resolved
LAMMPS/Moltemplate input decks into kUPS. The importer reads the numeric LAMMPS
files generated by Moltemplate and writes `resolved.yaml`, a stable handoff file
for later bonded, nonbonded, electrostatic, and dynamics milestones.

No kUPS potential is evaluated here. The important result is that the LAMMPS
source data can be represented without loss: IDs, topology, style declarations,
charge overrides, coefficient tables, unit conversions, and special-pair graph
classes are all visible to the checker.

Run the local reproduction command from the repository root:

```bash
PYTHONPATH=src python -m kups.polymerization.lammps_import \
  --source external/lammps_oplss/moltemplate_oplsaa_tiny \
  --out examples/polyremization/lammps_to_kusp/moltemplate_oplsaa_tiny_1/resolved.yaml \
  --report examples/polyremization/lammps_to_kusp/moltemplate_oplsaa_tiny_1/REPORT.md
```
